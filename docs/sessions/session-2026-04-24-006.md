## Session Notes

### Patterns established / reinforced this session

- **Three-pass evaluator convergence is the default for substantive bundles.** D8 is 10 sub-items + cross-cutting palette tokens; two fix-packs were needed to close everything. Pattern: don't expect Pass-1 to be clean for any substantive bundle; budget 2-3 passes with progressively smaller fix-packs (Pass-1: 17 files, +260/-50; Pass-2: 5 files, +69/-7; Pass-3: zero changes). The fix-pack commit message naming convention (`fix(10d): D8 Pass-N evaluator fix-pack`) makes the iteration legible in git log.
- **`data-*` semantic attributes are the right test addressability for state-driven styles.** `[data-consent-row][data-consent-state="granted"]` survives DOM restructuring and pins the load-bearing semantic, which is stronger than `getByText` regex assertions on user-visible copy. Used across the Pass-1 + Pass-2 D8.j fix-packs (data-routing-state, data-consent-state, data-stage-variant, data-destination-state). Worth carrying forward to D7's anonymous_id wiring (e.g. `data-anonymous-id` on the Overview portals-and-state block, `data-id-source="cookie"|"derived"`).
- **Doc reconciliation belongs in the same fix-pack as the deliverable.** Pass-1 fix-pack reconciled REQUIREMENTS.md D8.f (path + count discrepancy) alongside the deliverable closure rather than leaving it for a follow-up. Pass-2 fix-pack reconciled PHASE_STATUS.md D8.g + D8.j descriptions alongside the test pins. Pattern: when an evaluator surfaces doc drift, fix the docs in the same commit as the closure of the related issue. Avoids the "spec drift waiting room" anti-pattern.
- **Acceptable-as-is Minors get inline-documented rationale, not silent dismissal.** Pass-2 Tech Minor #3 (EventDetail/EventTimeline sent-styling asymmetry) was defensible per the reviewer; the fix-pack added an 11-line comment block explaining the deliberate volume difference rather than just leaving the disagreement implicit. Future contributors get the design memory inline rather than having to re-derive it.
- **Visual image identification is faster than name-based inference for asset mappings.** The D8.f image-to-product mapping pairing was opaque at the filename level (DSC + UUID + mixed-case names with no semantic hint). Visually inspecting each webp via the Read tool gave a confident mapping for 5/6 pairs in ~2 minutes. Pattern carried forward: when N unnamed assets need to map to N catalogue items, look at them rather than guessing from filenames.

### Architecture decisions

- **D8.j semantic tokens (`u-accept` / `u-deny`) added to the overlay Tailwind palette rather than inlined hex.** `tailwind.config.ts` carries the green (#8FBF7A) + red (#D9725B) tokens that match the demo banner pulse + data-quality readout already in use. Using tokens means a future palette tweak (e.g. swapping `#D9725B` for a slightly desaturated red) propagates across all five recoloured surfaces from one place.
- **D8.j `×` glyph distribution.** Glyph retained on consent-row status tags (where it's the only non-colour signal) and on EventDetail + NarrativeFlow surfaces (same rationale — single-glyph + colour, no strike-through). Dropped on Consent destination pills where `line-through` + red was already redundant. The asymmetry is intentional: glyphs supplement colour where the strike-through isn't available; on pills the strike-through replaces the glyph.
- **EventDetail vs EventTimeline `sent` styling deliberate volume difference.** Documented inline at `event-timeline.tsx:20-29`. Row chip (Timeline) keeps neutral body text + `u-accept`-tinted border only — visitor's eye stays on the event_name + page_path that carry row identity. Detail-panel badge (EventDetail) goes full `text-u-accept` because it's primary content when selected.
