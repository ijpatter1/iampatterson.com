### Bug Fixes During Deployment
1. **Pub/Sub push endpoint placeholder**, setup script used literal `XXXXXXXXXX` placeholder because `EVENT_STREAM_URL` wasn't set. Fixed script to fail loudly if env var missing (`9ec2892`)
2. **sGTM sandboxed JS limitations**, `Date`, `isConsentGranted`, `data` (reserved), `require('gtmOnSuccess')` are all unavailable. Rewrote template to use `makeString(getTimestampMillis())`, event data consent fields (`x-ga-cs-*`), `data.gtmOnSuccess()`, and `var` declarations
3. **Double base64 encoding**, manual test encoded payload before publish; Pub/Sub encodes again in push envelope. Fixed by publishing raw JSON
4. **Multi-instance routing**, Pub/Sub push hits random Cloud Run instances; SSE connection on different instance. Fixed with `--max-instances=1` (sufficient for portfolio site)
5. **Session ID field mapping**, GA4 sGTM client remaps `session_id` to `ga_session_id`. Updated tag to read `eventData.ga_session_id` first (`20340b9`)
6. **Next.js build failure on Vercel**, `infrastructure/` TypeScript files (express imports) compiled by Next.js. Fixed by excluding `infrastructure` from `tsconfig.json` (`0a5c914`)

## Key Learnings (for future sessions)
- sGTM sandboxed JS has no `Date`, `Math`, `isConsentGranted` (web-only), or `require('gtmOnSuccess')`. Use `makeString()`, `x-ga-cs-*` event data, and `data.gtmOnSuccess()` respectively
- GA4's sGTM client remaps custom event parameters: `session_id` → `ga_session_id`
- Pub/Sub push doesn't respect Cloud Run session affinity (push comes from Google infra, not browser)
- `gcloud pubsub topics publish --message` auto-base64-encodes in the push envelope, don't pre-encode
