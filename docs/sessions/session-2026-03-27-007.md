### Event Deduplication
- **Root cause**: React Strict Mode (`reactStrictMode: true`) double-fires `useEffect`, causing duplicate `page_view` events (`7ff2084`)
- **Fix**: Added `firedRef` guard in `RouteTracker` to skip duplicate fires for same pathname
- **Safety net**: Client-side dedup in `useEventStream` by `event_name + timestamp + page_path` (`ebf9f03`)

### Session ID for Cookieless Pings
- **Root cause**: GA4 remaps `session_id` ‚Üí `ga_session_id`. For cookieless pings (consent denied/withdrawn), GA4 overwrites this with its own numeric timestamp ID. Cloud Run can't route events to the correct SSE connection (`110a994`)
- **Fix**: Added `iap_session_id` parameter that GA4 won't remap. sGTM tag reads this first
- **Key learning**: Site already runs Advanced Consent Mode (consent defaults load before GTM). GA4 sends cookieless pings even when consent is denied ‚Äî transport is NOT blocked. The direct beacon workaround was removed as unnecessary

### GTM Container Updates
- Web container import (`web-container-import.json`) updated with: `iap_source`, `iap_session_id`, `consent_analytics`/`consent_marketing`/`consent_preferences` in shared event settings variable
- All GA4 event tags inherit these via shared settings ‚Äî no per-tag config needed
- Import via Overwrite (not Merge) to avoid duplicate tags

### Cloud Run Deployment Note
- `--max-instances=1` is required after every redeploy: `gcloud run services update event-stream --region us-central1 --max-instances=1`
- Pub/Sub push hits random instances; SSE connection is on one instance. Without this constraint, events can't be routed

## Key Learnings

1. **GA4 strips underscore-prefixed params** ‚Äî use `iap_source` not `_iap`
2. **GA4 remaps `session_id` to `ga_session_id`** and overwrites it on cookieless pings ‚Äî use a custom param name like `iap_session_id`
3. **Advanced Consent Mode is already active** if `gtag('consent', 'default', ...)` loads before GTM. Cookiebot GTM template is NOT required
4. **`x-ga-cs-*` fields** are populated by the sGTM GA4 client from `gcs`/`gcd` parameters, but custom data layer fields (`consent_analytics` etc.) are more reliable since they're under our control
5. **React Strict Mode** double-fires `useEffect` ‚Äî guard event-firing effects with refs
6. **Cloud Run `--max-instances=1`** resets on every redeploy ‚Äî must be re-applied
7. **GTM container import: Overwrite not Merge** ‚Äî Merge creates duplicate `_import_1` tags
