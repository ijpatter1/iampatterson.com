## Session Notes

### Pattern established: unconditional-close gate invariant

Both D2's `landingResolvedRef` and D1's NavHint state machine use the same architectural shape: when an effect emits conditionally on a state resolution, close the gate on the resolution itself, NOT inside the emission branch. Closing only inside the emission branch leaves a hole for later state changes to re-enter the effect and trigger spurious emissions. Pass 1 D2 was the Major dual-fire bug produced by exactly this anti-pattern. The pattern applies anywhere a `useEffect` emits conditionally, it's generalizable. Inline comments at both sites name the invariant.

### Pattern established: prototype-chain-safe allowlists

Any user-input-keyed lookup against a `Record<string, T>` object literal is vulnerable to prototype-chain inheritance. `?rebuild=toString` returns `Object.prototype.toString`. Fix: build the map with `Object.create(null)` (null-prototype) and guard lookups with `Object.hasOwn()`. This generalizes to any user-routed enum lookup (URL params, form fields, JSON body keys). Inline comment at `demos-section.tsx` names the failure mode so future maps get the same treatment.

### Pattern established: Suspense isolation for `useSearchParams()`

`useSearchParams()` forces a client-side rendering bailout on its caller. Isolating the hook in a leaf component wrapped in `<Suspense fallback={null}>` preserves static generation of the outer page. The null fallback is correct because the query param isn't observable during SSG anyway. Applies to any route that needs to read search params on an otherwise-static page.
