## Context

Session 019 produced `infrastructure/metabase/setup-domain.sh`, fully idempotent, creates the 7 LB components (static IP, Google-managed SSL cert, serverless NEG, backend service, URL map, HTTPS proxy, forwarding rule), prints the DNS A record, then polls cert status. Evaluator cleared it with the NEG-match and cert-failure hardening applied in `c5dea66`.

The only non-scriptable step is **creating the DNS A record at GoDaddy**, GoDaddy is the registrar per repo memory, and we don't have automated access to it. Google's managed cert will not provision until that A record resolves.

**Prerequisite:** Task 3 must be done (`docs/manual/task-2026-04-17-001.sh`) so the Cloud Run backend exists for the NEG to target.

## Steps

### 1. Run the setup script (creates 7 LB components, prints DNS instruction)

```bash
cd /workspace/infrastructure/metabase
./setup-domain.sh 2>&1 | tee /tmp/metabase-task5.log
```

The script will pause after creating components 1–7 and print a line like:

```
==> CREATE THIS DNS RECORD AT YOUR REGISTRAR:
        bi.iampatterson.com.  A  <STATIC_IP>
    Then return here. This script will poll cert status every 30s.
```

Copy the static IP.

### 2. Create the A record at GoDaddy

Navigate: <https://dcc.godaddy.com/control/portfolio/iampatterson.com/settings?tab=dns>

Add record:

- **Type:** A
- **Name:** `bi`
- **Value:** the static IP from step 1
- **TTL:** 1 hour (3600 seconds)

Save. GoDaddy's default propagation is typically a few minutes.

### 3. Verify DNS propagation before resuming

```bash
dig bi.iampatterson.com +short
# expect: <STATIC_IP> (matching the value from step 1)
```

If `dig` doesn't return the IP yet, wait a few minutes and try again. Don't proceed until the A record resolves, Google's cert provisioning will fail otherwise.

### 4. Resume the script (it polls cert status)

The `setup-domain.sh` run from step 1 is still waiting. Return to that terminal, it will poll every 30 seconds and exit when the cert is ACTIVE (typically 15–60 minutes after DNS resolves). If the terminal was closed, just re-run the script, it's idempotent and steps 1–7 skip, polling resumes.

### 5. Verify once cert is ACTIVE

```bash
curl -sI https://bi.iampatterson.com/api/health | head -3
# expect: HTTP/2 200 (before IAP attaches in Task 6, the endpoint responds directly).
# After Task 6 this becomes 302 to accounts.google.com, that's correct.

gcloud compute ssl-certificates describe metabase-cert \
  --project=iampatterson --format='value(managed.status,managed.domainStatus)'
# expect: ACTIVE, bi.iampatterson.com=ACTIVE

URL=$(gcloud run services describe metabase \
  --region=us-central1 --project=iampatterson --format='value(status.url)')
curl -sI "${URL}/api/health" | head -1
# expect: 404/403, .run.app stays locked (ingress=internal-and-cloud-load-balancing).
```
