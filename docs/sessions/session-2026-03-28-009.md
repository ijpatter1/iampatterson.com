### Transport Debugging & Fixes
- **`/mp/collect` returns 400 on sGTM**: Discovered that sGTM's GA4 client doesn't serve the Measurement Protocol JSON endpoint. Tested against Google's endpoint (204 OK) confirming payload was valid but sGTM doesn't support this path
- **Rewrote transport to `/g/collect` format**: URL-encoded query parameters matching gtag.js format (v=2, tid, cid, en, dl, dt, ep.*, epn.*). POST still returned 400
- **GET works, POST doesn't**: sGTM GA4 client only accepts GET requests at `/g/collect`. Switched to GET with all params in query string, 200 OK
- **Removed API secret requirement**: `/g/collect` format doesn't need `api_secret`, simplifying deployment (no GA4 Admin setup needed)

### Performance Fixes
- **OOM on 18-month backfill**: Generator was building ~500K events in memory before sending. Added `streamingBackfill()` that generates and sends one day at a time, keeping memory constant
- **Slow sequential sends**: Was sending one HTTP request per event (~1400/min, ~6 hours for full backfill). Added concurrent sending with `Promise.allSettled` (30 parallel requests), ~20-30x speedup

## Key Learnings

1. **sGTM GA4 client only accepts GET at `/g/collect`** with URL-encoded query params. POST with same params in body returns 400. `/mp/collect` (Measurement Protocol JSON) is not served by sGTM at all
2. **No API secret needed for `/g/collect`**, events are accepted in the same format as browser traffic
3. **GA4 protocol param format**: `v=2`, `tid=`, `cid=`, `en=`, `dl=`, `dt=`, `ep.*` (string params), `epn.*` (numeric params), `_s=`, `_et=`, `_ss=`, `sid=`
4. **Streaming is essential for backfill**, 500K events in memory OOMs at 1Gi. Day-by-day streaming keeps memory constant
5. **Concurrent requests are essential**, sequential sends take 6+ hours. 30 concurrent requests bring it to ~15-20 minutes
6. **GA4 doesn't backdate events**, all events show as today's date in GA4 reports. BigQuery stores the simulated timestamp in `ep.timestamp` for Dataform to use
