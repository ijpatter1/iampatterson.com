## Session Notes

- **All commits went directly to `main`** instead of a `phase/6-demo-front-ends` branch. This was a process error — should have branched before starting. The code is correct but bypassed the phase branch → PR → merge workflow.
- **Homepage demo CTAs were disabled** from Phase 1 as placeholders. This wasn't caught until the user reported they couldn't see demos on the live site. Lesson: always check the entry points when activating deferred features.
- **The `ga4 - shared_event_settings` variable** in the web GTM container handles all base params (iap_source, timestamp, session_id, page_path, page_title, consent_*). Phase 6 tags only need event-specific parameters in their own `eventSettingsTable`. This pattern keeps tags DRY.
- **sGTM simulated tags can't be fully imported** via JSON alone — sGTM custom templates require sandboxed JavaScript that must be registered as a template in the sGTM workspace. The import creates placeholder tags that need template attachment post-import.
