## Session Notes

### Pattern established: derive-from-schema as a triangulated invariant

This session codified the "derive from schema day one" rule in a way future sessions can safely build on:

1. **Runtime source of truth:** `DATA_LAYER_EVENT_NAMES` const array in `src/lib/events/schema.ts`. Consumers iterate this array instead of maintaining parallel allow-lists.
2. **Compile-time cross-check:** `_AssertEventNamesInSync` sentinel forces the runtime array and the `DataLayerEvent` union to stay in lockstep via tuple-wrapped `[T] extends [U]` conditional types (avoids distributive behavior). Build fails on drift.
3. **Test-level triangulation:** `schema.test.ts` exhaustive `Record<DataLayerEvent['event'], true>` literal catches runtime/union drift at test time. The DO-NOT-REPLACE marker in the test explains why the hand-list literal is the type-level exhaustiveness mechanism and must not be "simplified" to a runtime-array iteration.

Three consumers now derive from the same source: `useDataLayerEvents`, `ambient-bubbles`, and Session State deliverable 4 (when built). Future event additions require editing `DATA_LAYER_EVENT_NAMES` only; consumers absorb the addition automatically.

### Pattern established: closed-enum discipline for typed discriminators

`CtaLocation` as a closed 14-value union (7 nav-adjacent + 4 editorial + 3 legacy) demonstrates the pattern for tightening a `string` discriminator across the codebase:

1. Define the closed union in the schema module with JSDoc explaining the categorization.
2. Tighten the interface field type (`ClickCtaEvent.cta_location: CtaLocation` not `string`).
3. Audit every existing call site to normalize values (snake_case migration across 9 sites; `Tier.num` narrowing; demos-section template → static property).
4. Test that call sites and type system agree (`cta-link.test.tsx` now uses valid enum values; closed-enum test asserts the 7 nav-adjacent values).
5. Legacy values stay in the union with explicit JSDoc attribution until their emitting components are deleted (deliverable 7 removes `subscription_dashboard` + `demo_card_subscription` + `demo_card_leadgen`; deliverable 6 may also touch `demo_card_*`).
