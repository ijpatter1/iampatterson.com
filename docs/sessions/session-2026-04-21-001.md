### User-reported follow-up fixes (post-F8)

**Refresh persistence audit** (commit `af3a7ad`). User re-reported "page refresh resets session data in UI." Full audit: SessionPulse event count, LiveStrip CONSENT/SESSION fields, Pipeline footnote, Timeline tab, Consent tab were all reading live `window.dataLayer` / `useLiveEvents` and showed empty state on refresh. Fixed by:
- SessionPulse: `count = Σ sessionState.events_fired[name]` with fallback to live buffer
- LiveStrip: `consentLabel(sessionState, events)` prefers `sessionState.consent_snapshot`
- LiveStrip: session id prefers `sessionState.session_id` (reconciled against cookie)
- New `src/lib/events/timeline-buffer.ts` ring-buffer module with `load/save` helpers (100-event cap)

- Full audit table in the commit message documents every stateful UI surface + what was/wasn't changed (AmbientBubbles deliberately preserves only-new-events behavior). +6 tests.

**Hydration-error revert** (commit `0751a34`). The `useState(initialLoad)` + `useState(() => loadTimelineBuffer())` eager-loads introduced a hydration mismatch: SSR produced null/[] values, client first render produced hydrated values, and SSR'd consumers (SessionPulse, LiveStrip, pipeline-editorial, ride-along) rendered different text content → React "Text content does not match server-rendered HTML" error dialog in dev. Reverted both lazy initializers to `useState(null)` / `useState([])`; load now happens in post-mount useEffect (client-only, never runs on server). Persistence writes remain intact. Trade-off accepted: brief flash of default values (one paint frame) on refresh, but no error dialog and clean hydration.

## Session Notes

### Pattern established: surface value = transmitted value

The F8 Important #10 fix (ride-along denominator matching Overview denominator via `RENDERABLE_EVENT_NAMES`) established a principle: if a visitor sees `X/Y` on screen, the payload transmitted should carry `{numerator: X, denominator: Y}`. Conflicting values between UI and payload breaks the site's measurement-transparency conceit. Applies anywhere else a ride-along pattern is added — chrome readouts, contact-form state, URL params.

### Pattern established: metric classification matters more than completeness

The final-pass NavHint fix (removing `click_session_pulse` from `dismissal_mode`) followed the "conversions vs abandonments must be separable in BI" principle. Enumerating every possible event is not the same as enumerating the useful ones. When a dismissal enum captures conversions alongside abandonments, the dismissal metric answers no product question. Applies to future analytics additions: every enum value should justify its existence by answering a distinct BI question.

### Pattern established: persistence must reach every surface

The refresh-persistence audit (commit `af3a7ad`) scanned every UI component that reads session-scoped or live-event data, mapped each to its source of truth, and ensured every SSR'd consumer's source survived refresh. The audit table in the commit message is reusable — copy it forward when a new UI surface consumes session data.

### Decision: SSR hydration safety > zero-flash UX

The eager-load approach (`useState(initialLoad)` / `useState(() => loadTimelineBuffer())`) would eliminate the flash-of-empty-state on refresh, but produced hydration text-content mismatches that React flagged as runtime errors in dev. Reverted to post-mount useEffect loading. Brief flash (one paint frame) is the accepted trade-off. If zero-flash becomes a priority, the cleanest path forward is `useSyncExternalStore` with separate `getServerSnapshot` and `getSnapshot` — explicitly SSR-safe for subscription-style stores. Not worth the complexity here.

### Decision: historical log entries stay frozen

`docs/sessions/*.md` session artifacts, `docs/uat/phase-9e-uat-output.md`, and `docs/PHASE_STATUS.md` historical deliverable log entries all describe what was true when they were written (pre-F1 vocabulary, older enum values, past commit-message content). Rewriting them to reflect post-close-out naming would falsify the record. Headers + "Current" sections stay fresh; log entries stay frozen. The F8 drift sweeps deliberately left historical log entries alone.
