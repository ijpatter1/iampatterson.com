## Steps

### 1. Configure OAuth consent screen (one-time, UI only)

Navigate: <https://console.cloud.google.com/apis/credentials/consent?project=iampatterson>

Fill in:

- **User Type:** Internal
- **App name:** `iampatterson BI`
- **User support email:** your Google account (ian@tunameltsmyheart.com)
- **Scopes:** leave defaults (email, profile, openid)
- **Developer contact info:** your email

Save. The project now has an OAuth brand `setup-iap.sh` can reference.

### 2. Edit the allowlist if needed

Before running the script, confirm the allowlist in `infrastructure/metabase/setup-iap.sh` near line 50:

```bash
ALLOWLIST=(
  "user:ian@tunameltsmyheart.com"
  # add more lines like "user:someone@example.com"
)
```

Additions are applied on each run. Removals are deliberately manual (see Task 6 section in `infrastructure/metabase/README.md`).

### 3. Run the IAP setup script

```bash
cd /workspace/infrastructure/metabase
./setup-iap.sh 2>&1 | tee /tmp/metabase-task6.log
```

What it does:

1. Creates OAuth client `metabase-iap-client` (idempotent by display name).
2. Stores client ID + secret in Secret Manager (`metabase-iap-client-id`, `metabase-iap-client-secret`).
3. Enables IAP on `metabase-backend`.
4. Grants `roles/iap.httpsResourceAccessor` to each allowlist member.

### 4. Verify the IAP gate is live

```bash
# Should redirect to Google SSO, proves IAP intercepted the request.
curl -sI https://bi.iampatterson.com/ | head -3
# expect:
#   HTTP/2 302
#   location: https://accounts.google.com/...

# IAP enabled on backend?
gcloud compute backend-services describe metabase-backend \
  --global --project=iampatterson --format='value(iap.enabled)'
# expect: True

# Allowlist bound?
gcloud iap web get-iam-policy \
  --resource-type=backend-services --service=metabase-backend \
  --project=iampatterson --format='value(bindings.members)'
# expect: all ALLOWLIST members
```

### 5. Browser check, the "can I actually get in?" test

Open <https://bi.iampatterson.com/> in a browser.

- Logged in as an allowlisted Google account → Google login page → Metabase first-run wizard
- Logged in (incognito) as a non-allowlisted account → "You don't have access"

If the 302 redirect doesn't happen, set Status to `blocked`. Common causes:

- OAuth consent screen not fully saved, revisit the Console URL and check for validation errors.
- IAP bindings propagation, can take up to 7 min after the script exits. Re-run verify step 4 after waiting.
- Backend service doesn't report `iap.enabled=True`, the script's `gcloud compute backend-services update --iap` call failed (likely a permission gap on `roles/iap.admin`).
