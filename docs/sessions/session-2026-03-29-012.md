## Session Notes

- **Dataform can't read subdirectories** — requires `dataform.json` at repo root. Solved with a dedicated `dataform` branch + GitHub Action sync. This pattern works but required debugging the Action (a `git checkout dataform -- .` line was silently undoing the sync).
- **BigQuery `CREATE TABLE ... CLUSTER BY` is incompatible with `ORDER BY` in the query** — hit this twice (data_dictionary, campaign_taxonomy_validation). Dataform compiles the config's `clusterBy` into the DDL, so any query with ORDER BY must not have clusterBy in config.
- **AI.CLASSIFY uses the default Vertex AI connection** — no need to specify `CONNECTION` explicitly. Explicit syntax caused parse errors.
- **URL encoding from `/g/collect`** — all string values sent via GET params have `+` for spaces and potentially `%XX` encoding. Must decode in staging layer. The `url_decode.js` helper handles this with correct nesting order (%25 first, + last).
- **sGTM `ignoreUnknownValues: true` silently drops columns** — root cause of missing demo params in the original schema. Always verify raw table schema matches what the transport sends.
- **Cloud Run won't kill instances with in-flight requests** — `--no-traffic` stops new requests but old instances stay alive. Must delete service to force-kill.
- **Returning visitors require separate `client_id` from `session_id`** — the GA4 `cid` param must persist across sessions for LTV and MTA to work. ClientPool with 30% return rate achieves this.
