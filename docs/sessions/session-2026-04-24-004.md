## Session Notes

### Patterns established / reinforced this session

- **Homogeneous cluster → one branch, multiple commits, no evaluator mid-cluster.** The D8 bundle is 10 independent small items; running the evaluator per-cluster vs per-sub-phase vs per-item is a tuning decision. Chose per-sub-phase (10d-wide) since each item has direct test-pin coverage and the failure modes are limited to "did I touch the right surface + walk the right pins." Product-reviewer surface-voice scrutiny is more load-bearing than tech review for this kind of work; queued up for after the full D8 lands.

- **Class-presence pins for CSS-variable-driven visuals.** The D8.c pipeline CTA persimmon change is a pure CSS + Tailwind-class shift with no JS state. jsdom doesn't compute `--bleed`-driven transitions, so the only Jest-testable regression guard is asserting the presence/absence of specific Tailwind classes on the button element. The pin uses both positive (`toContain('bg-accent-current')`) and negative (`not.toContain('bg-paper')`) assertions so a half-rewrite that merges old + new can't sneak through.

- **Commit-level scope discipline in deletion work (D8.e).** The ProofSection deletion touched 4 files (component, test, homepage import/render, one stale comment in an adjacent test). Each reference was surfaced via `grep -rn "ProofSection|proof-section|proof-card"` before the deletion, not after; zero-residue sweep discipline carries from 10c. Next-session callers can `grep` the same patterns to verify no ghosts remain.

- **Plural→singular copy audits surface tests that use the phrase as a fixture, not as a feature pin.** `cta-link.test.tsx` used "Explore the demos" as a test-fixture example for JSX-children extraction. The pattern: when you're changing a piece of user-facing copy, grep the tests for the phrase but distinguish between tests that *pin* the phrase (must move with the source edit) and tests that *happen to use it as a fixture* (can stay but should move too, to avoid grep false positives). Both were updated this session.

### Architecture decisions

- **D8.c palette choice: `var(--accent)` (site persimmon #EA5F2A), not `#ffa400` (bleed-layer orange).** The button's at-rest colour anchors on the site's primary accent token, not the bleed layer's hotter orange. The bleed ramp still transitions *into* `#ffa400` at peak, so the visual escalation from the at-rest persimmon → bleed-active hotter orange → peak jitter block still reads as a progression. Using `var(--accent)` also means the Tuna Shop `--accent` override (`#c4703a`, per `globals.css:43`) would carry through if the pipeline-section ever rendered inside a Tuna Shop container — which it doesn't today, but the token-driven approach is portable by construction.

- **D8.d emission URL → deep-link.** The `trackClickNav` second arg is `linkUrl` per the `src/lib/events/track.ts:70` signature. Honest thing is to emit the actual URL the visitor is being sent to, not the generic parent route. Next session should consider whether the ClickNav fan-out downstream (funnel reporting in Metabase) wants to distinguish `/services` visits from `/services#tier-02` visits — if yes, the hash-including URLs give that granularity; if no, it's still honest-by-construction and can be trimmed at the analytics-layer.
