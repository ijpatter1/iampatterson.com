### Patterns established / reinforced this session

- **Copy-only branch pattern.** D3-D5 is a text-change-only branch with zero functional changes (one schema union addition being the only non-text edit). The test-pin-walk discipline (every string change looks for its pin, walks it forward in the same commit) is the regression gate; `npm run build` catches TypeScript + Tailwind issues; the usual Functionality / Architecture evaluator dimensions are low-signal on copy-only work. Product reviewer is the primary reviewer; technical evaluator spot-checks pin integrity + schema change + orphaned references.
- **Workshop coupling (resolved).** The "See your session" button-label workshop coupled tightly to the Final CTA headline ("See your session first. Then hire me.") which echoed it. Rather than changing the button text, R2 worked the "making the invisible visible" framing into the hero prose ("Measurement is usually invisible; here, you can watch it live. Your session is the portfolio."), which earns the button click without orphaning the headline echo. Pattern worth remembering: when a button label is under-earning, consider whether the surrounding prose can earn it before reaching for a rename.
- **Visitor-scoped vs infrastructure-scoped thesis.** The hero closer moved from "The instrumentation is the portfolio" to "Your session is the portfolio" during R2. Pattern: subject-choice on thesis statements. Visitor-subject is usually stronger for buyer-facing copy because the reader has something their own in the sentence; infrastructure-subject lands as a pitch about Ian's work. Applied once in 10c; worth watching for other places where Ian's copy has infrastructure-subject thesis that could be visitor-scoped.
- **Possessive-genitive parse traps.** "Every click of your session" illustrated the pattern: possessive-of-X noun phrases where X is a polysemous noun ("session") create parse ambiguity. Prefer preposition-of-location ("in your session") when grounding a new noun, and anchor explicit nouns rather than chains of "it" references across registers (italic → prose).
- **Pre-existing issue flagging.** When a voice audit surfaces a pre-existing issue outside the PR's rewrite scope (services hero comma splice), flag it in the handoff rather than expanding the commit. Keeps the PR atomic + gives the next session a fresh audit lane.
- **Scope-discipline self-check for hide decisions.** The 10b D1a `HIDDEN_FROM_COVERAGE` call for `web_vital` assumed an "interaction-only" scope for the Overview coverage grid that the site never actually codified. When making hide / exclude calls for surfaces that already render non-interaction events (`nav_hint_shown`, `coverage_milestone`, `*_tab_view`), check whether the exclusion criterion is consistent with what's already rendered — or you'll end up with arbitrary-looking omissions that need reversing under user review. The session-scope framing the user imposed ("everything that flows through the stack during the visitor's time, not only events they trigger explicitly") is worth carrying forward to future hide-decision reviews.
