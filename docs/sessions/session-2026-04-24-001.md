### Architecture decisions

- **`HIDDEN_FROM_COVERAGE` for `web_vital`:** CWV fires 5 events per page load regardless of visitor action. Including it in the Overview chip grid denominator would trivially light 5/N slots on first paint and destroy the depth-of-exploration signal the meter is for. Event still flows through the standard pipeline + lands in BigQuery columns; hide is purely a visitor-surface concern.
- **`capture-cwv-baseline.sh` runs BOTH mobile + desktop presets** because the numbers tell a different story on each. Mobile is "where the optimization work lives"; desktop is "what prospects see." Both captured; both written to `docs/perf/` as the machine-diffable artifacts against future optimization work.
- **Gitignore `lighthouse-*.report.html`** so committed artifacts stay JSON-only (human-readable HTML is bulky; JSON is the diff-able source of truth).
- **D6 render-counter via `useEffect` not render-body:** satisfies `react-hooks/immutability` (mutation is post-commit side-effect, not render work). Pass-2 fix-pack added `process.env.NODE_ENV` guard so Next.js constant-folds to dead code in production.
- **Test-only global counter over Profiler/spy instrumentation:** React.Profiler reports per-commit subtree timing, not per-component render count. Spying on internal components requires exporting them just for tests. Global counter with a `typeof` guard + top-level `afterEach` delete is the minimum-invasive pattern that passes the `react-hooks/immutability` rule and pins the memo short-circuit claim directly.
