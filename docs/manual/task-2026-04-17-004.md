### 3. Retrieve the BigQuery SA key for step 7

Run this in a terminal when you're on the Metabase BigQuery data-source form:

```bash
gcloud secrets versions access latest --secret=metabase-bq-sa-key
```

This prints the JSON key to stdout. Copy it and paste into Metabase's **Service account JSON** field. Do not save the key to disk — the clipboard is volatile, and Metabase encrypts the pasted value with `MB_ENCRYPTION_SECRET_KEY` the moment you save the data source.

Field values for step 7:

- **Database type:** BigQuery
- **Display name:** `iampatterson marts`
- **Project ID:** `iampatterson`
- **Service account JSON:** paste from the command above
- **Datasets:** dropdown `Inclusion`, comma-separated field `iampatterson_marts`
- **Advanced options:** leave defaults

### 4. Verification (the "can I query real data?" test)

After step 8 in the runbook:

1. **+ New** (top-right) → **Question**
2. **Pick your starting data** → `iampatterson marts` → `mart_campaign_performance`
3. **Visualize**
4. Rows should come back within a few seconds. If rows return, the pipeline is working end-to-end: IAP → Metabase → BigQuery (dataset-scoped SA) → `iampatterson_marts`.
