## Session Notes

### Patterns established / reinforced this session

- **Subset-invariant tests check the wrong direction for typo drift.** Both Pass-1 (sitemap product URLs) and Pass-2 (sitemap static URLs) closed the same shape of test gap: `arrayContaining([...])` lets silent drops through. Symmetric tightening is `length === expected.length` + per-element `toContain` loop. Memory pattern reinforced.

- **Honest narrowing in audit docs.** Both the mobile matrix doc and security review have explicit "what this can't measure / can't certify" sections that name the falsifying signals + dispatch them as manual tasks where appropriate. Pattern: when shipping an audit artifact, write the limitations section first; the limitations are what determine whether the audit is honest or theater.

- **Half-finished accessibility patterns are worse than no pattern.** The Pass-1 fix-pack added `role="tablist"` without finishing the tab pattern (`role="tab"` + `aria-selected` + `role="tabpanel"` + `aria-controls`). Pass-2 evaluator caught it; closure attempt broke 24 tests; correct call was to revert the partial pattern entirely. Memory pattern: don't add ARIA roles unless you're closing the full pattern in the same change.

- **Source-of-truth divergence is fragile.** `SITE_DESCRIPTION` in root layout vs. JSON-LD Organization description disagreed because the JSON-LD got rewritten in one fix-pack and the layout constant was missed. Pass-2 closure included a comment block above the constant flagging the cross-file invariant. Pattern: when one of two consistent surfaces gets rewritten, grep for callers + add a comment naming the invariant.

- **Spec test gates on E2E_ENABLED=1 + project metadata for matrix-style coverage.** The mobile-matrix spec follows the pattern from `ecommerce-sessionpulse.spec.ts`: gate on env flag, use Playwright project metadata (`mobileLabel`, `mobileChromeBoost`) so per-project assertions are data-driven not hardcoded. The capture script + matrix doc + manual task triple is the new pattern for coverage-style evaluation deliverables.

### Architecture decisions

- **`/demo/ecommerce/layout.tsx` Server-Component lift via `EcomProviders`.** Next.js metadata exports forbidden on `'use client'` modules. `CartProvider` requires client; lifted to `src/components/demo/ecommerce/ecom-providers.tsx` so the layout itself is server. This is the canonical pattern for any future route that needs both `metadata` export + client-side providers.

- **OG image as edge-runtime `ImageResponse` route, not committed PNG.** Trade-off: regenerable + always reflects current code at the cost of edge-runtime font fetching per render. Font-bundling verified via `.next/server/edge/assets/instrument-serif*.ttf`. Pattern: any future "default + per-route override" image surface should use this same shape.

- **Sitemap excludes session-scoped routes; robots disallows them as belt-and-suspenders.** Cart, checkout, confirmation, `/contact/thanks` are intentionally not indexable. The exclusion list lives in two places (`sitemap.ts` filtering + `robots.ts` Disallow) so a future drop in either place is recoverable from the other.

- **JSON-LD as builder functions, not static JSON.** `organizationJsonLd()` + `personJsonLd()` return plain JSON-serializable objects. Inlined as separate `<script type="application/ld+json">` tags rather than a single `@graph` wrapper — the two-script form is more widely supported by rich-result parsers.
