## Architecture Notes

The overlay system uses a shell/content separation pattern:
- **Shell components** (`DesktopOverlayShell`, `MobileBottomSheetShell`) handle layout, positioning, and platform-specific behavior (fixed sidebar vs bottom sheet)
- **OverlayContent** handles all view logic (tabs, timeline/narrative/consent modes, event selection/detail)
- **OverlayPanel** orchestrates: reads `useEventStream`, pipes events through `useFilteredEvents`, renders content inside both shells
- Both shells exist in the DOM simultaneously; CSS media queries (md:flex / md:hidden) control which is visible
