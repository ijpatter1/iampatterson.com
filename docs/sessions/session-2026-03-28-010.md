## Session Notes

- **sGTM only accepts GET at `/g/collect`** with URL-encoded query params. POST returns 400. `/mp/collect` (Measurement Protocol JSON) is not served at all. This is a key learning for any future server-to-sGTM integration.
- **GA4 `/g/collect` param format:** `v=2`, `tid=`, `cid=`, `en=`, `dl=`, `dt=`, `ep.*` (strings), `epn.*` (numbers), `_s=`, `_et=`, `_ss=`, `sid=`. No API secret needed.
- **Streaming is essential for long backfills** — in-memory accumulation OOMs at ~500K events with 1Gi.
- **Concurrent requests critical** — sequential HTTP takes 6+ hours; 30 concurrent brings it to ~15-20 min.
- **GA4 doesn't backdate events** — all events show as today in GA4 reports. BigQuery gets `ep.timestamp` (the simulated time) for Dataform to use as the logical timestamp.
- **The backfill needs to be re-run** after deploying the timestamp fix. The existing data in BigQuery lacks `ep.timestamp`.
