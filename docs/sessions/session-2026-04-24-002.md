## Session Notes

### Patterns established / reinforced this session

- **Time-filtered DML to sidestep BQ streaming buffer.** When a UPDATE/DELETE on a streaming-fed table fails with `"would affect rows in the streaming buffer"`, narrow the WHERE clause with `received_timestamp < TIMESTAMP_SUB(NOW, INTERVAL N HOUR)` and run iteratively (start at 2hr, then 1hr, etc.). Applies whenever sGTM-streamed events need a remap. Reusable pattern; documented in cat-content scrub MD apply progression as a playbook.
- **CREATE OR REPLACE TABLE on partitioned tables drops the partition spec.** Any table-level recreation needs explicit `PARTITION BY x CLUSTER BY y` clauses preserved. Discovered via the streaming-buffer flush attempt failing. Forced the 3-step (CREATE OR REPLACE staging → DELETE window → INSERT deduped → DROP staging) approach for the 6-day overlap dedup.
- **MERGE on UNNEST(struct array) requires explicit INSERT cols + STRING + PARSE_DATE for date params.** `INSERT ROW` shorthand fails because BQ sees the source as a single STRUCT-typed column. STRUCT-param `date` field as 'DATE' type doesn't reliably auto-cast. Pattern documented in `bq-insert.ts:67-76` + `bq-insert.test.ts` regex-pins on the load-bearing fragments.
- **Generator state.isRunning band-aid via schedule stagger.** Cloud Scheduler `:00/:20/:40` per model gives 13-17 min headroom over 3-7 min `/generate` runtime. Acceptable until a model regresses past 20 min runtime; documented as deliberate Phase 11 ops deferral in REQUIREMENTS scope amendment.

### Architecture decisions

- **D2 + ops follow-ups absorbed into 10c rather than deferred to Phase 11** because each manifests as a *visibly dishonest* number on the demo dashboard prospects see — which is the literal failure mode 10c exists to prevent. Documented as scope amendment in REQUIREMENTS Phase 10c block.
- **FORBIDDEN list (`['cat', 'feline', 'kitten']`) chosen over allowlist for brand-vocab pin** because allowlist would require enumerating every plausible English word that's not off-brand. Closed list is clearer about *what's banned*; widened over time as new bug classes surface.
- **6-day overlap dedup via 3-step DELETE+re-INSERT-after-staging** chosen because BQ DELETE doesn't support window functions in WHERE; CREATE OR REPLACE on the live table loses partition spec. Trade-off: 4 short query jobs vs 1 atomic statement, but preserves all schema metadata.
- **Whisker Works retained** despite cat-adjacent vibe (chiweenies have whiskers; vibrissae aren't cat-exclusive; B2B partner brand names don't need 100% dog-coding). Documented decision rather than unstated judgment call.
