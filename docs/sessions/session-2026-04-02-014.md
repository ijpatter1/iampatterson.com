## Session Notes

- **GTM API rate limit:** The Tag Manager API has a per-user per-minute query limit. When creating 34+ entities sequentially, you'll hit it. The workaround is to wait 60s between batches. The deploy script doesn't handle this automatically.
- **sGTM CONTAINER_CONFIG format:** The correct format is obtained from the GTM API snippet endpoint (`containers/{id}:snippet`), NOT manually constructed. The format is `id=GTM-XXXX&env=N&auth=XXXX` (base64 encoded). Previous attempts with `aContainerId`, `aEnv`, `aAuthCode` or numeric IDs all failed.
- **sGTM sandboxed JS restrictions:** No global `JSON` — must `require('JSON')`. Must declare `___SERVER_PERMISSIONS___` for `logging` and `read_event_data`. The `address_info` nested object in templates was removed to simplify and avoid potential sandbox issues.
- **Service account permissions accumulated this session:** `roles/iam.serviceAccountUser` (Cloud Run deploy), `roles/logging.privateLogViewer` (read logs), `roles/run.admin` (domain mappings + IAM). These are on `claude-code-sandbox@iampatterson.iam.gserviceaccount.com`.
