## Session Notes

### Environment fix: arm64 Node required

Session started with 105/105 test suites failing at `0 tests ran` because `@next/swc-darwin-x64` is not published for Next 14.2.34+ (only `darwin-arm64`), and the resident Node at `/usr/local/bin/node` is a Mach-O x86_64 binary running under Rosetta 2 on an arm64 Mac. Next 14 fell back to Wasm SWC, which `next build` tolerates but Jest's 8 parallel workers crash on with "Jest worker encountered 4 child process exceptions." The user installed arm64 Node at `/opt/homebrew/bin/node` (v20.20.2) mid-session; clean reinstall under that Node fixed the baseline to 1164 passing in ~3s. Every Bash call this session prepended `export PATH="/opt/homebrew/bin:$PATH"` since the shell doesn't persist between tool invocations. Captured as feedback memory (`env_arm64_node_path.md`) so future sessions automatically prepend.

### Pattern: REQUIREMENTS.md scoping stale under mechanical migration

REQUIREMENTS.md D1 named **seven** files for the async-Request-API codemod. The actual codemod output modified **one**. Consider softening to 'approximately seven files identified at scoping time; codemod is the authoritative pass' when starting 10a work"`). The warning was correct; lesson for future mechanical-migration phases: write the scoping fence as "the codemod is authoritative" rather than pinning a file-count.
